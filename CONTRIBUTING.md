# Contributing to JSONtapose

## Process

1. Fork the repo
2. Create a branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit your changes
5. Push to your fork
6. Open a Pull Request

## Pull Request Rules

1. All PRs must be made against the `main` branch
2. PRs require at least one approval from a CODEOWNER
3. All PRs must be merged using squash merge
4. Follow the PR template when creating your pull request

## Development Rules

1. Write all code and comments in English
2. Follow the project's code style (TypeScript, ESLint, Prettier)
3. Write tests for new features
4. Keep components small and focused
5. Document complex logic

## Questions?

Open an issue if you have any questions about contributing.
